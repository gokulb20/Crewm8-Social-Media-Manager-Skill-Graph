# Podcast to Clips

Distill podcast and video interview transcripts into platform-native social clips. A single 45-60 minute episode should yield 5-8 social posts across platforms, not just one "new episode out" announcement.

## When to Use

- New podcast episode or YouTube interview published
- Founder was a guest on someone else's podcast
- Older episode that performed well deserves a second push
- Building a "clips library" for ongoing social content
- Turning one long-form asset into a week's worth of posts

## Quick Reference

| Podcast Segment | Best Clip Format | Best Platform |
|----------------|-----------------|--------------|
| Hot take / controversial opinion | Quote tweet or text-over-video clip | X, TikTok |
| Data point or surprising stat | Quote card + stat-led post | X, LinkedIn |
| Personal story | Video clip (30-60s) + caption | IG, TikTok |
| Framework or system | Carousel or thread | LinkedIn, X |
| Practical tip / how-to | Short video (15-30s) or text post | TikTok, IG, X |
| Funny or human moment | Video clip (15-30s) with light caption | All platforms |

## Procedure

1. Obtain the full transcript or watch the full episode. Note timestamps for every moment that:
   - Made you pause or think differently
   - Contained a specific number or data point
   - Told a concrete story
   - Stated something contrarian or controversial
   - Offered a framework or step-by-step process
   - Was genuinely funny or human

2. For each timestamped moment, determine the best social format:
   - Quote tweet style: pull the exact quote, add 1-2 lines of context or reaction
   - Video clip: note the exact time range (15-90 seconds), write the text overlay hook
   - Carousel/post: if it's a framework, draft the slide structure
   - Thread: if there are multiple connected insights, build a mini-thread (3-5 tweets)

3. For video clips specifically:
   - Clip length: 15-30s for TikTok/Reels, 30-90s for X video, 60s max for LinkedIn
   - Always include captions/subtitles
   - Write the text overlay hook that appears in the first 1.5 seconds
   - Write the caption that accompanies the clip (different per platform)

4. For each clip asset, draft:
   - The hook (text overlay or caption opening line)
   - The caption/body text
   - The CTA ("Full episode link in bio," "Follow for more clips")
   - Relevant hashtags per platform

5. Organize the clips into a posting schedule:
   - Don't post all clips the same day
   - Spread across the week following episode release
   - Best clip first (highest engagement bait within 24 hours of episode drop)

6. Include attribution:
   - If the founder was a guest: tag the host's account in the first clip
   - Link to the full episode in every clip's caption or final tweet

## Output Format

```markdown
# Podcast Clips: [Episode Title / Guest Name]

## Episode Source
[Podcast name + episode link + timestamp of full episode]

## Clip Inventory

### Clip 1: [Topic / Quote]
**Timestamp:** [MM:SS - MM:SS]
**Format:** [Video clip / Quote tweet / Thread / Carousel]
**Platform priority:** [Primary platform + secondary]
**Hook / Text overlay:** [Text]
**Caption / Body:** [Full caption text]
**CTA:** [CTA text]
**Hashtags:** [Platform-appropriate set]

### Clip 2: ...
...

## Posting Schedule
- Day 1 (episode release day): [Clip X on platform Y — highest engagement bait]
- Day 2: [Clip X on platform Y]
- Day 3: [Clip X on platform Y]
...
```

## Pitfalls

- Extracting clips the host finds interesting but the audience doesn't (different curation lens)
- Video clips without captions/subtitles (unwatchable for most social consumption)
- Posting all clips at once (floods followers, reduces per-clip engagement, wastes content)
- Using the same caption for every clip ("new episode out!") instead of clip-specific hooks
- Clips that require context from the full episode to make sense (each should stand alone)

## Verification

For each clip: if you saw this in your feed without knowing the podcast, would it make sense and make you want to engage? If any clip's hook is "new episode" or "we talked about X" — rewrite to a content-first hook.
