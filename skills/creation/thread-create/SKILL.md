# Thread Create

Build an X thread that earns every read and every retweet. Threads are the highest-leverage format on X for education, storytelling, and authority-building. A weak thread is worse than no thread — it trains the algorithm that your content isn't worth surfacing.

## When to Use

- Converting a blog post or newsletter into a thread
- Sharing a framework, system, or hard-won insight
- Product launch narrative
- Contrarian take that needs more than 280 characters to land
- Case study or results breakdown
- Content calendar calls for an education/pillar thread

## Quick Reference

| Thread Element | Rule |
|---------------|------|
| Length | 6-10 tweets optimal. Under 5 = "long tweet." Over 12 = loses readers. |
| Hook tweet | First 3 words must stop the scroll. No links. No hashtags. |
| Body tweets | One idea per tweet. Use line breaks aggressively. |
| Pacing | Escalate insight/value with each tweet. Don't front-load. |
| Final tweet | Clear CTA. Link (if any) goes here. |
| Visuals | 1 image/chart per thread max. Place in tweet 2-4. |
| Auto-plug | Add link to first tweet AFTER engagement threshold is met (see `thread-structure` skill). |

## Procedure

1. Identify the thread angle from your source material:
   - Blog post: extract top 5-7 insights, one per tweet
   - Product launch: Problem → Old way → Our way → Proof → CTA
   - Case study: Challenge → What they tried → What worked → Results → Lesson
   - Contrarian take: Common belief → Why it's wrong → What actually works → Examples
   - How-to guide: Step-by-step, one step per tweet, tools/suggestions
   - Revenue update: Number → Breakdown → What worked → What failed → What's next

2. Write the hook tweet (tweet 1):
   - Lead with your strongest word or number
   - 3 styles that work:
     - The Promise: "I analyzed 100 SaaS pricing pages. Found 3 patterns"
     - The Contrarian: "Everyone tells you to niche down. Here's when you shouldn't"
     - The Story Hook: "We hit $10K MRR. Then almost lost it all"
   - Avoid: "Here's what I learned about X" (weak), "A thread 🧵" (wastes characters)
   - No links in hook tweet
   - Use the 🧵 emoji only if it naturally fits; don't force it

3. Write the body (tweets 2 through N-1):
   - Tweet 2: Establish credibility or context. Why should they trust you?
   - Tweets 3-7: Core content. One specific, concrete idea per tweet.
   - Escalate: each tweet should feel like it's building toward something bigger
   - Use line breaks for visual rhythm. Dense text blocks get scrolled past.
   - Interleave: pattern-interrupt with a question, a bold statement, or data
   - If using an image/chart, place it in tweet 2-4 (shows up in preview on timeline)

4. Write the final tweet:
   - Summarize the key takeaway in one line
   - Clear CTA: "Follow @handle for more," "Newsletter link below," "What's your experience? Reply"
   - Link (if any) goes here, not in first tweet
   - Make it satisfying — threads that trail off feel unfinished

5. Thread quality checklist:
   - Hook strength: does tweet 1 make you NEED to click "Show this thread"?
   - Value density: no filler, no fluff, every tweet earns its place
   - Flow: does each tweet naturally lead to the next?
   - CTA: is the last tweet clear and actionable?
   - Brand voice: does it match the company's tone?
   - Fact check: any claims need sources or data?

6. Apply auto-plug strategy:
   - Do NOT include the link in the initial post
   - After engagement threshold is met (X likes or Y minutes), reply to first tweet with link
   - See `thread-structure` skill for detailed auto-plug rules

## Output Format

```markdown
🧵 THREAD: [Title/topic]

Tweet 1: [full text]
Tweet 2: [full text]
Tweet 3: [full text]
...
Final Tweet: [full text + CTA]

---
Hook strength: [brief assessment]
Expected engagement: [low/medium/high based on topic and hook]
Best posting time: [recommendation from cadence-timing]
Follow-up post: [suggest a standalone tweet that could go live 2 hours after thread]
Visual recommendation: [image/chart/quote card idea + where to place it]
Auto-plug link: [link to add after engagement threshold, if applicable]
```

## Platform Notes

- X threads: max 280 chars per tweet, no markdown, use line breaks
- Emojis: use strategically, not decoratively
- Hashtags: 0-1 per thread (organic reach > hashtag discovery on X)
- Links: avoid in first tweet (reduces reach). Put in final tweet or auto-plug after engagement
- Thread length: 6-10 is optimal. Under 5 is a "long tweet." Over 12 loses readers.

## Pitfalls

- Weak hooks: "Here's what I learned about X" is NOT a hook
- Wall of text: X is visual whitespace — use line breaks liberally
- Over-promising: threads should deliver value, not clickbait
- Auto-plugging too early: wait for engagement before adding links
- Posting threads at terrible times: avoid 3 AM audience time
- Threads that don't end: every thread needs a clear conclusion + CTA
- Being too generic: specific > broad, examples > abstract claims
- First tweet over 280 characters (breaks thread unfolding in timeline)

## Verification

Read the hook tweet alone. Would you click "Show this thread"? Count filler words across all tweets — cut them. Does tweet 2 deliver on the promise of tweet 1? Does the final tweet feel like a real ending or did it just... stop?
